# 03 并发续行持久化与去重策略

## 开发目标

解决多个 subagent 在短时间内并发结束时，continuation scope 被覆盖、误去重或部分丢失的问题。

本里程碑的重点是：

1. 并发 ended 不丢 continuation。
2. 重复 ended 仍然可去重。
3. 父会话即使分多轮处理，也能最终消费完全部 continuation。

## 开发思路和步骤

### 总体思路

把 continuation 的持久化粒度固定到 run 级，保证：

1. 一个 ended 至少对应一个可追踪 continuation scope。
2. 不同 run 的 continuation 不会互相覆盖。
3. 同一 run 的重复 ended 不会无限生成重复 scope。

### 开发步骤

1. 明确 continuation 的主去重粒度：
   - 至少包含 `dag_id + run_id`
   - outcome 仅作为补充维度，不应导致跨 run 混淆
2. 强化 pending continuation 的可重放语义：
   - 父会话只消费第一条时，剩余 continuation 仍保留
3. 收紧重复 ended 的幂等策略：
   - 同一 run 重复 ended 不重复产出无限 continuation
4. 让并发 completed / ready 事件与 continuation scope 对齐：
   - 能继续按 run 追踪
   - 不因批量总结而丢失 run 级信息
5. 保证父会话可逐步消费：
   - 一次没处理完，不影响后续继续推进 DAG

## 测试方案

1. 两个 task-dag run 并发 ended：
   - 检查是否保留两条独立 continuation scope
2. 父会话只消费其中一个 scope：
   - 检查另一条仍然存在
3. 同一 run 重复 ended：
   - 检查 continuation 不无限重复
4. 并发 ended 导致多个下游 ready：
   - 检查后续 `task_dag_continue` 仍能逐步推进
5. 一个普通 subagent ended 与一个 task-dag run ended 并发：
   - 检查只保留 task-dag continuation

## 测试预期结果

1. 并发 ended 不会只保留一条 continuation。
2. 不同 run 的 continuation 不会互相覆盖。
3. 同一 run 的重复 ended 会被正确去重。
4. 父会话可以分多轮消费 continuation，而不会漏推进 DAG。
