# 02 兼容层上下文统一

## 问题描述

部分兼容层工具仍然直接使用：

```ts
args?.agent?.id || args?.agentId || 'main'
```

这会导致：

- 在非 `main` agent 下可能读写错误目录
- 在同一 agent 下存在多个 DAG 时无法精确定位 `dag_id`
- 与新的 `setExecutionContext()` / binding / continuation 体系脱节

## 修改目标

让仍保留的兼容工具统一走新的上下文解析逻辑，并支持 `dag_id`。

重点工具：

- `task_dag_modify`
- `task_dag_subtask_create`
- `task_dag_subtask_list`
- `task_dag_context`
- `task_dag_resume`
- `task_dag_logs`
- `task_dag_set_doc`
- `task_dag_get_doc`

## 修改方案

1. 把这些工具统一切到 `setExecutionContext()`
2. 为这些工具补 `dag_id` 参数
3. 保持旧参数兼容，但以新上下文解析为主
4. 增加多 DAG / 非 `main` agent 的回归测试

## 测试方案

1. 在同一测试内创建两个不同 DAG
2. 对第二个 DAG 使用兼容工具读取和修改
3. 断言不会误读到第一个 DAG
4. 再切换到非 `main` agent，重复至少一个兼容工具场景

## 预期结果

- 兼容工具不会再默认落回 `main/default`
- 多 DAG / 非 `main` agent 路径下行为与新工具一致
- 旧工具不再成为上下文污染点
