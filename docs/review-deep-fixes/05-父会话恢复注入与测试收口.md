# 05 父会话恢复注入与测试收口

## 修复目标

- 降低父会话恢复对“模型读懂 `[task-dag-resume]` 文本并自觉调用工具”的依赖。

## 修复思路

- 保留 `subagent_ended -> sessions_send` 的主动唤醒。
- 新增 `before_prompt_build` hook：
  - 根据当前 `sessionKey` 查找 pending `resume_requested`
  - 在父会话新一轮 prompt 前注入明确 continuation 指令
- 这样 `sessions_send` 负责唤醒，`before_prompt_build` 负责把下一步协议稳定前推到系统层。

## 测试方案

- 验证父会话 `sessionKey` 存在待处理 `resume_requested` 时，`before_prompt_build` 返回 `prependContext`。
- 验证没有待处理恢复事件时，`before_prompt_build` 不注入任何内容。
- 复跑现有 `subagent_ended -> continue` 回归，确认新注入机制不破坏旧链路。

## 预期结果

- 父会话被唤醒后的下一轮，会在 prompt 层明确收到 `task_dag_continue` 指令。
- 恢复链路对文本消息本身的脆弱依赖下降，整体更接近插件驱动。
