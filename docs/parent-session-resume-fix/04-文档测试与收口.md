# 04 文档测试与收口

## 修改目标

把 README、SKILL、测试矩阵和调试文档更新到“插件主动维护父会话继续机制”的新模型。

## 问题所在

- 当前文档还把“父会话下一轮调用 `task_dag_continue`”描述得过于理想化
- 没有明确写出：插件会在 ended hook 后主动唤醒 requester session

## 执行方案

   - 补充 requester session 注册表
   - 补充 ended hook -> `sessions_send` -> parent continuation 链路
2. 更新 SKILL
   - 强调优先走插件高层协议
   - 明确插件会主动唤醒父 session
3. 扩展测试矩阵
   - spawn / hook / completion / requester resume
   - direct delivery 共存时的去重
4. 收口临时日志和实现细节

1. `npm run build`
2. `node test_runner.mjs`
3. `bash test-all.sh`
4. 重点检查：
   - hook 原始 shape 测试
   - requester session 注册表测试
