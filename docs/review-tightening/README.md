# 纯新架构收紧修复

本目录对应一轮纯新架构 code review 之后的收紧修复，目标是把插件进一步收口到单一事实来源、显式上下文、硬状态机约束，减少多 agent、多 DAG 和 hook 时序下的漂移空间。

包含 5 个修复项：

- `01-上下文解析强约束.md`
- `02-状态机迁移硬校验.md`
- `03-删除平行元数据系统.md`
- `04-spawn事件单一来源.md`
- `05-移除旧waiter注册表.md`

统一执行原则：

- `binding/session-run/pending-event` 是唯一运行时事实来源。
- 不再通过全局残留状态猜测 agent/dag。
- 非法状态迁移直接拒绝，不再“尽量帮忙执行”。
- hook 与 tool 的生命周期职责单一，不双写相同语义事件。
- 删除与新架构不一致的旧元数据和注册表。

统一测试要求：

- `npm run build`
- `node test_runner.mjs`

统一预期：

- 全量测试通过。
- 旧平行数据结构不再参与主执行路径。
- 多 agent / 多 DAG 场景下，所有关键工具都需要显式上下文或可从确定性绑定中恢复上下文。
