# 03 删除平行元数据系统

## 问题所在

兼容接口虽然已删，但旧元数据系统仍存在：

- `agent-mappings.json`
- `session-mappings.json`
- `session-hierarchy.json`

这些文件与 `task-bindings.json`、`session-runs.json`、`pending-events.jsonl` 平行存在，且 hook 仍在依赖其中一部分，最终会形成双事实来源。

## 修复思路

- 删除不再被新架构需要的 agent mapping 能力。
- 删除 session mapping 与 hierarchy 的持久化文件和读写路径。
- hook 改为仅依赖：
  - event 自带字段
  - `session-runs.json`
  - `task-bindings.json`
- 与通知恢复相关的旧逻辑如果仍保留，改为通过 DAG 和 binding 推导，不再通过旧 mapping 推导。

## 测试

- 新增测试：`subagent_spawned` 之后无需旧 session mapping 仍能通过 session run 与 binding 恢复 ended 上下文。
- 新增测试：项目目录中不再创建旧 mapping 文件。
- 回归测试：hook 正常 spawn / ended / orphan 行为不退化。

## 测试预期结果

- 旧 mapping 文件不再落盘。
- hook 只依赖新 binding/session-run/event 体系即可完成上下文恢复和任务收口。
