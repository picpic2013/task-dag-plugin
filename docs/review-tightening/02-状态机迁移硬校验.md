# 02 状态机迁移硬校验

## 问题所在

当前 `claim/spawn/assign/complete/fail` 可以在几乎任意 task 状态下直接执行，导致：

- `blocked` 任务可被直接 claim
- 已终态任务可被重新 spawn / assign
- 非执行中任务可被直接 complete / fail

这会破坏 DAG 依赖和执行协议。

## 修复思路

- 引入统一的状态校验函数：
  - `claim` 只允许 `ready` 或部分可恢复的 `running`
  - `spawn` / `assign` 只允许 `ready`
  - `progress` 只允许 `running` / `waiting_subagent`
  - `complete` / `fail` 只允许 `running` / `waiting_subagent`
- 对 session 多任务场景，`assign` 只能把 `ready` 任务加入已有 run。
- 非法迁移返回结构化错误，包含当前状态和允许状态。

## 测试

- 新增测试：`blocked` 任务 claim 失败。
- 新增测试：`done` 任务 spawn 失败。
- 新增测试：`done` 任务 assign 失败。
- 新增测试：`ready` 任务 complete 失败。
- 保留现有正常路径测试，确保合法迁移仍然通过。

## 测试预期结果

- 非法状态迁移全部被拒绝。
- 合法的 `ready -> running -> done/failed`、`ready -> waiting_subagent -> done/failed` 路径保持正常。
