# 04 删除任务时清理运行时元数据

## 修复目标

- 在删除 task 时同步清理绑定、session run、pending event、requester scope 和任务文档。
- 避免 DAG 结构删除后留下运行时垃圾，污染后续 hook / reconcile / continue。

## 修复思路

- 为 bindings 层增加定向清理能力：
  - 删除指定 task 的 binding
  - 从 session run 的 `active_task_ids` 中移除 task
  - 若 session run 已无任务，删除该 run 及 `by_session` 索引
  - 删除与 task 直接相关的 pending event
- 为 requester session scope 增加移除 task 的操作；若 scope 清空则删除 scope。
- `dag.removeTask()` 在递归删除子任务时也删除对应文档。
- `task_dag_modify(action="remove")` 调用清理流程。

## 测试方案

- 新增回归：删除一个等待中的 task 后，其 binding、pending event、scope 记录被清理。
- 新增回归：删除带子任务的父 task 时，递归子任务的运行时元数据也被清理。

## 预期结果

- 被删除的 task 不会在后续 continuation、ended hook、reconcile 中再次出现。
- 文档文件和运行时元数据保持一致。
