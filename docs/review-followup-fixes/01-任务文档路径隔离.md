# 01 任务文档路径隔离

## 修复目标

- 让任务文档按 `agent_id + dag_id` 隔离，避免不同 DAG 中同名 task 文档互相覆盖。
- 修正 `main` agent 文档路径与 DAG 主数据路径规则不一致的问题。

## 修复思路

- 将文档目录调整为：
  - `main -> ~/.openclaw/workspace/tasks/{dag_id}/docs/`
  - `other -> ~/.openclaw/workspace-{agent_id}/tasks/{dag_id}/docs/`
- `setTaskDoc()` 的文件名继续按 `taskId.md`，但因目录按 DAG 隔离，不再发生跨 DAG 覆盖。
- 删除 task 时同步删除对应文档文件。

## 测试方案

- 新增回归：两个 DAG 使用同一个 `task_id`，分别写文档，验证路径不同、内容不覆盖。
- 新增回归：`main` agent 下文档应写入 `workspace/tasks/{dag_id}/docs`。

## 预期结果

- 同名 task 在不同 DAG 下的文档互不影响。
- `main` agent 文档路径与主数据路径一致。
