# 04 统一读写工具参数契约

## 修复目标

- 让读工具和写工具使用相同的定位参数体系。
- 减少模型因接口不一致而误用工具。

## 修复思路

- 为所有读写工具统一支持：
  - `agent_id`
  - `dag_id`
  - `task_id` 或 `task_ids`
- 补全写工具 schema 中缺失的 `agent_id`。
- 统一返回体中的 `agent_id`、`dag_id`、`task_id` 信息。
- 统一错误返回结构，区分参数错误、上下文错误、状态机错误、业务不存在错误。

## 测试方案

1. 使用同一组参数分别调用：
   - `task_dag_get`
   - `task_dag_claim`
   - `task_dag_progress`
   - `task_dag_complete`
2. 检查工具 schema 中是否都暴露了 `agent_id`。
3. 检查返回结果中是否都带 `agent_id` 和 `dag_id`。

## 测试预期结果

- 读写工具的参数使用方式一致。
- 同一条 `chexie` 场景调用链可以稳定推进。
