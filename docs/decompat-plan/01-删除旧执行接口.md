# 01 删除旧执行接口

## 任务目标

删除仍然鼓励旧执行模型的外部工具接口，只保留新执行协议。

## 开发任务

1. 删除 `task_dag_wait`
2. 删除 `task_dag_notify`
3. 删除 `task_dag_update`
4. 删除 README / SKILL 中对这些接口的主路径描述

## 执行细则

### 删除项

- 工具注册：
  - `task_dag_wait`
  - `task_dag_notify`
  - `task_dag_update`
- 对应 helper：
  - 如果仅为这些工具服务，也一并删除

### 替代路径

- `task_dag_wait` -> `task_dag_continue`
- `task_dag_notify` -> `task_dag_progress` / hook / pending events
- `task_dag_update` -> `task_dag_claim` / `task_dag_progress` / `task_dag_complete` / `task_dag_fail`

## 测试方案

1. 现有测试不得再调用以上 3 个工具
2. 保留并强化以下主路径测试：
   - `claim/progress/complete/fail`
   - `spawn/assign`
   - `continue/reconcile`
3. 通过工具注册测试确认以上 3 个名字不再存在

## 预期结果

- 外部无法再通过这些旧接口进入旧执行模型
- 新协议成为唯一执行主路径
