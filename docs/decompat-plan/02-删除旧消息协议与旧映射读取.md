# 02 删除旧消息协议与旧映射读取

## 任务目标

删除依赖模型自觉遵守文本协议和旧 `session -> task` 兼容读取的路径。

## 开发任务

1. 删除 `message:received` 下的 `TASK_COMPLETE/...` 文本协议解析
2. 删除 `parseTaskMessage()` / `handleTaskMessage()` 等旧逻辑
3. 删除 `getTaskBySession()` / `getTasksBySession()` 中对旧兼容读取的主逻辑依赖
4. 让 hook 与 continuation 只依赖 binding / session-run / pending-event

## 执行细则

### 删除项

- `hooks.ts`
  - `parseTaskMessage`
  - `handleTaskMessage`
  - `message:received` hook
- `agent.ts`
  - `getTaskBySession()` 的旧 fallback
  - `getTasksBySession()` 的旧 fallback

### 保留项

- session hierarchy
- session mapping 文件
  - 仅当 hook 或调试仍需要最小辅助信息时保留

## 测试方案

1. hook 测试必须只依赖：
   - `subagent_spawned`
   - `subagent_ended`
   - bindings / events
2. 删除所有对 `TASK_COMPLETE/...` 文本协议的测试预期
3. 验证：
   - 单任务 session 收尾
   - 多任务 session 收尾
   - orphan 场景

## 预期结果

- hook 收尾不再依赖模型输出特定字符串
- 运行时主路径完全结构化
